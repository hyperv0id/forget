**英文名：** The Forget  
**Slogan：** Forget for Get (遗忘是为了获得 / 得失之间)  
**版本：** v0.05.1（Core Loop Revision + Clarifications）  
**设计日期：** 2026-01-10  
**修订日期：** 2026-01-12  
**原作背景：** 《明日方舟》岁系列角色 - 望（Wang）  
**核心身份：** 第181枚神识碎片（黑子）

---

## 0. v0.05.1 改动摘要（相对 v0.05）

本小版本不改变资源名词与系统框架，只统一三处“口径可能被理解成不同规则”的描述，以便后续实现与玩家理解一致：

1. **名望为负时的行为触发：** 依然结算，但“好感度上升”效率降低（而不是完全不触发）。
2. **模仿与好感度：** 模仿牌本身只会直接降低（或不变）好感度；但模仿牌产生的战斗事件仍可能间接触发行为规则，从而带来好感度上升（名望为负时同样会被折损）。
3. **污染：** 污染定义为战斗内塞入的垃圾牌/状态牌压力，不跨战斗滞留；体面态可用名望清理以缓解本场战斗节奏。

---

## 0. v0.05 改动摘要（相对 v0.04）

本版本的目标：让【模仿】成为“必须围绕它构筑”的体系核心，并让【名望】与【好感度】之间形成更清晰的循环（参考《王权》(Reigns) 的“资源被迫平衡”体验），同时不增加新的资源名词。

**核心改动：**

1. **开局选择 3 个对象：** 本局只围绕这 3 位对象结算【好感度】；与对象绑定的【模仿】牌奖励只从这 3 位中出现，降低“抓不到关键牌导致系统不可达”的风险。
2. **废除眷恋牌：** v0.05 不再存在“直接提高好感度”的专门牌型。好感度只由【名望】与战斗中的出牌行为/战斗事件决定（例：打技能、获得格挡、击杀等）。
3. **名望为负时的模仿代价：** 名望为负时依然可以打出【模仿】牌，但**每打出一张模仿牌额外失去 1 点生命**（Lose HP，不计护甲）。其它牌照常，不额外扣血。

## 一、核心设计愿景 (Core Vision)

### 1.1 高概念
玩家扮演的不是“望”本人，而是为了屠神而切分出的**神识碎片**。卡牌是记忆，战斗是落子；你必须在“赢下当下”与“写坏自己”之间不断权衡。

### 1.2 核心冲突（保持不变，但闭环更明确）

* **资源冲突：** 你需要【名望】维持体面（清理污染/维持节奏），但又需要通过【权谋】拉扯名望去换取战斗胜率。
* **关系冲突：** 你需要【模仿】兄弟姐妹的权能来获胜，但每一次模仿都会触发“关系账本”的变化。
* **自我冲突：** 当名望为负时，你依然可以模仿，但代价会更直接回流到【自我】（HP）。

## 二、基础资源系统 (Resource System)

### 2.1 【自我】 (Self / HP)
* **定义：** 传统生命值。
* **特殊性：** 当名望与关系都不足以支付代价时，HP 会成为最后的支付手段；v0.05 明确让“名望为负时的模仿”直接吃 HP。

### 2.2 【名望】 (Reputation)

名望是一条**跨战斗的全局环境数值**，允许进入负值。它决定你是否能用名望清理手牌中的【污染】，并决定【模仿】在“关系账本”与“HP 账本”之间更偏向哪一边付费。

* **名望 > 0（体面态 / High Light）：**
    * **环境特性：** 可以消耗名望，清理手牌中的【污染】（诅咒/状态）。
    * **清理操作：** 玩家可主动消耗 `N` 点名望，将手牌中的【污染】（诅咒/状态）移除本场战斗。
    * **好感度结算：** 体面态下，好感度的“行为触发”按**完整效率**结算；无耻态下仍会结算但效率降低（见 2.3），强化“体面=更能维持关系”的直觉。

* **名望 < 0（无耻态 / Low Light）：**
    * **环境特性：** 无法用名望清理【污染】，【污染】牌更容易在本场战斗内反复形成节奏压迫。
    * **模仿代价：** 仍可打出【模仿】牌，但**每打出一张模仿牌额外失去 1 点生命**（Lose HP）。
    * **数值修正（沿用 v0.04 方向）：** 可保留“全局伤害加成”作为无耻态的直接收益，但需要设置上限/分段，避免鼓励玩家长期贴边（调参阶段决定）。
    * **好感度结算：** 名望为负时，战斗中的出牌行为/战斗事件**依然结算到好感度，但“好感度上升”效率降低**；名望为正时按完整效率结算（见 2.3）。

> 说明：名望的数值范围与阈值仍然属于“调参项”，但 v0.05 强调**名望翻转带来的玩法翻转**必须清晰可感知（UI/音画提示）而不是藏在数字里。

### 2.3 【好感度】 (Affinity)

* **定义：** 按对象分别结算的隐藏数值。本版本中，“对象”默认只包含本局开局选择的 3 位（见第三章）。
* **可见性：** UI 不显示具体数值，只通过卡面变化与动态提示让玩家能做出决策（见 3.3）。

#### 2.3.1 好感度的经济循环（v0.05.x 口径）

* **消耗端：** 打出【模仿：X】会改变【好感度：X】（默认是降低；具体降低幅度/阈值触发由单卡决定）。【模仿】本身**不会直接带来好感度上升**。
* **回复端：** v0.05 不通过“专门的修复牌”来提高好感度。好感度主要通过**名望状态 + 出牌行为/战斗事件**来获得或失去（见 2.3.2）；其中“上升”在名望为负时依然可发生，但效率降低。

#### 2.3.2 行为触发：每位对象一套“你怎么打牌 → 我怎么给好感”

本节只定义规则口径，不固化数值。设计目标是：**玩家的出牌方式会自然把好感度推向你当前的构筑方向**，而不是额外塞一套“刷数值”的操作。
好感度的变化点全部挂在“打牌/战斗事件”的结算上，因此玩家体感应接近：**每次出牌都在改变关系走向**（只是改变幅度与触发对象不同）。

通用约束（建议）：
* **每位对象的触发每回合最多结算 1 次**（或有上限），避免被 0 费循环牌刷爆。
* 触发条件尽量与《杀戮尖塔》的常规出牌行为一致（打技能、叠甲、击杀），不引入新的操作方式。

示例（v0.05.x 默认口径）：
* **令 (Ling)：** 名望 > 0 时，打出技能牌 → 【好感度：令】+1（体现“逍遥/策略”）。
* **年 (Nian)：** 名望 > 0 时，获得格挡 → 【好感度：年】+1（体现“守护/铸造”）。
* **夕 (Dusk)：** 名望 > 0 时，有敌人死亡 → 【好感度：夕】+1（体现“抹除/清场”）。
* **重岳 (Chongyue)：** 名望 > 0 时，单回合打出第 3 张攻击牌 → 【好感度：重岳】+1（体现“武艺/连招”）。
* **黍 (Shu)：** 名望 > 0 时，打出能力牌 → 【好感度：黍】+1（体现“播种/投资”）。

> 重要：这些条件的目的不是“模拟人物性格”，而是把好感度变成**构筑反馈**：你越像技能流/护甲流/击杀流在打牌，你就越能“自然维持”对应对象的好感度。

> v0.05.1 补充口径：名望为负时，这些“行为触发”依然结算，但“好感度上升”效率降低（调参项决定折损系数/累积规则）。这允许玩家在无耻态维系关系，但更慢、更吃节奏与牌序质量。

### 2.4 累计反噬次数

* **定义：** Act 4 强度与结局判定的核心统计参数（过程累加制）。
* **计算口径（沿用 v0.04）：**
    * `累计反噬次数` = 本局游戏中，因好感度过低而触发“模仿不稳定/反噬”效果的累计总次数。
* **v0.05 强调点：** 好感度由名望与出牌行为决定，因此“你为了赢而不得不反复触发反噬”的过程，会更像你自己的选择，而不是单纯因为缺少修复手段导致的被动。

## 三、开局选择：本局只围绕 3 个对象结算

### 3.1 选择规则
游戏开始时（Neow 后的开局流程中），玩家从候选列表里**选择 3 位对象**（例如：令、年、夕等）。

* 本局只会生成这 3 位对象的【模仿】牌奖励（卡池限定）。
* 本局只会结算这 3 位对象的【好感度】与“模仿不稳定/反噬”（用于累计反噬次数）。

### 3.2 设计目的
* **降低随机性伤害：** 把“对象系统”从“抽到才算数”变成“开局就确定的构筑方向”。
* **让模仿成为体系核心：** 你不是“偶尔抓几张模仿牌”，而是从开局就围绕 3 个对象去构筑一整套牌序与资源循环。
* **方便调参：** 不同对象组合天然是不同流派入口，便于后续做强弱校准与卡池补齐。

### 3.3 UI 口径（不新增资源，只增强可读性）
* 在角色面板附近显示 3 个对象的小图标（或名称），用于提示“本局选择了谁”。
* 鼠标悬停图标时显示“该对象本回合触发条件”（例如：令=技能、年=格挡、夕=击杀）。
* 仍然不显示具体好感度数值，但【模仿】牌 Hover Tip 需要明确提示“当前关系区间将触发的附加代价”（沿用 v0.04 动态提示规则）。

## 四、卡牌系统：围绕【模仿】构筑的 4 类牌

| 类型 | 颜色/视觉 | 核心功能 | v0.05 对应资源逻辑 |
|:---|:---|:---|:---|
| **模仿 (Imitate)** | 渐变/多态 | 主要强度来源 | 结算【好感度】变化；低名望时额外 Lose HP；好感度越低越易触发不稳定/反噬并计入累计反噬次数 |
| **权谋 (Scheme)** | 暗金/灰色 | 推拉名望、换取窗口 | 消耗/产出名望，连接“体面清理污染”与“无耻爆发”；是 v0.05 的循环关键点 |
| **落子 (Move)** | 纯黑 | 终结技/禁忌爆发 | 大幅改变战斗胜负，但留下长期代价（扣名望上限、增加累计反噬次数、加污染等） |
| **污染 (Pollution)** | 墨黑/噪点 | 干扰手牌 | 体面时可用名望清理；无耻时作为倒计时与节奏压力 |

### 4.1 【模仿】在 v0.05 的“体系核心”口径

模仿不只是强牌，而是你整个角色的胜率来源，因此它必须满足两条口径：

1. **必须依靠其它牌才能发挥全部实力。**  
   “其它牌”主要指：权谋（提供名望窗口与节奏交换）、以及“出牌行为本身”（触发好感度变化）。

2. **模仿的代价必须能回到玩家可理解的账本上。**  
   v0.05 只使用现有三条账本：名望（是否能清理污染）、好感度（关系好坏）、自我（HP）。

### 4.2 【权谋】在 v0.05 的循环口径（参考《王权》）

权谋牌的设计目标不是“提供强度”，而是“迫使你在三条账本之间换来换去”，让玩家始终处于**资源不可能同时满**的状态。

建议的权谋牌方向（仅口径，不固化卡名与数值）：

* **推高名望：** 让你回到体面态、重新获得清理污染的能力与好感度行为触发，但通常要付出短期战斗代价（例如：弃牌/加污染/本回合输出下降）。
* **压低名望：** 让你进入无耻态，获得更直接的爆发窗口，但会带来清理污染不可用与模仿扣血的压力。
* **名望窗口：** 用卡牌在战斗中推拉名望，从而决定“本回合/本场战斗”是否还能通过出牌行为让好感度上升，形成可控的上坡路，避免系统锁死。

## 五、玩法阶段与进程 (Progression)

### 5.1 前期 (Act 1-2)：确定流派与学习循环
* 开局选择 3 个对象后，本局的“出牌方式”会自然偏向其中一两位（技能/护甲/击杀等）。
* 玩家学习用【权谋】制造名望窗口：体面态维持卡组、无耻态处理硬仗。
* 因为没有眷恋牌，“维持好感度”完全依赖名望与出牌行为，避免出现“抽不到修复牌就锁死”的构筑问题。

### 5.2 中期 (Act 3)：账本开始互相挤压
* 污染密度开始变得真实；清理污染与名望变得更珍贵。
* 玩家会因为某些战斗需求被迫更频繁地进入无耻态，从而把模仿代价压到 HP 上。
* 若依赖模仿但好感度维持失败，将开始更频繁触发不稳定/反噬并增加累计反噬次数。

### 5.3 后期 (Act 4)：镜像审判与终局结算
* **Act 4 精英：【望 - 空壳】** 与最终 Boss 的强度仍与累计反噬次数挂钩（沿用 v0.04 方向）。
* v0.05.x 下，累计反噬次数更像“你在资源挤压下做出的选择”而不是单纯 RNG；因此审判也更像“对你的账本的回收”。

## 六、结局口径的兼容说明（v0.05.1）

结局文本中出现的“【全员好感度】”在 v0.05.1 中默认解释为：**本局开局选择的 3 位对象**的好感度总体情况（而不是岁系全名单）。
同理，按“某对象好感度最高”判定的温情结局，也默认只在本局选择的对象范围内比较。

## 七、技术实现关键点 (Tech Checklist)

*   [ ] **RunState / Save：** 存储“本局选择的 3 位对象”，并写入存档（Save & Quit 不丢失）。
*   [ ] **ResourceManager：** 实现双向名望轴；名望 < 0 时的全局修正（若保留）要有分段/上限入口方便调参。
*   [ ] **ActionManager / UI：**
    *   清理操作交互：名望 > 0 时可用；名望不足/名望为负时要明确提示不可用原因。
    *   对象图标：展示本局 3 个对象以及其“本回合触发条件”的提示文本。
*   [ ] **AffinityManager：**
    *   存储 `Map<CharacterEnum, int>` 好感度（仅对本局选择对象生效）。
    *   行为触发监听：技能牌打出/获得格挡/击杀等事件 → 名望为正时按完整效率结算；名望为负时仍结算但上升效率降低（建议做“每回合结算上限”，并用可调参的折损系数/累积进度避免四舍五入造成“看似不触发”）。
    *   模仿结算：打出【模仿】时对对应对象好感度只做直接扣减（或不变），不做直接增加；随后由战斗事件触发的行为规则可间接带来好感度上升。模仿若触发不稳定/反噬则增加累计反噬次数。
*   [ ] **Mimic HP Cost：** 名望 < 0 时，打出【模仿】牌额外 `LoseHP(1)`（不计护甲），其它牌不受影响。
*   [ ] **Pollution：** 污染为战斗内塞入的垃圾牌/状态牌压力（不跨战斗滞留）；名望 > 0 时可用清理操作将其移出本场战斗。
*   [ ] **CardBase：**
    *   重写 `applyPowers()` / `calculateCardDamage()`，使【模仿】牌能读取 AffinityManager 并动态修改描述（Description）。
    *   动态卡面：根据好感度区间切换 img path，并与 Hover Tip 的红字代价保持一致。
